# 企业列表

## 1. 接口描述

* **接口说明** ：获取企业列表和资产信息，支持根据资产名字模糊查询
* **接口路径** ：asset/GetEnterpriseList
* **请求方式** ：POST

## 2. 输入参数

| 参数名称             | 类型                | 是否必填 | 描述                 |
|------------------|-------------------|------|--------------------|
| assetName        | string            | 否    | 数据资产中文名字             |
| assetEnName      | string            | 否    | 数据资产英文名字
| dataProductTypes | Array of [String] | 否    | 数据资产类型：0数据集 |

## 3. 输出参数

| **参数名称**       | **类型**                | **描述** |
|----------------|-----------------------|--------|
| enterpriseList | array of [Enterprise] | 企业信息   |

## 4. 示例

### （1）输入

```json
{
  "assertName": "资产名称"
}
```

### （2）输出

* 成功

```json
{
  "status": "success",
  "code": 200,
  "msg": null,
  "data": {
    "enterpriseList": [
      {
        "enterpriseName": "org1",
        "enterpriseAssetList": [
          {
            "assetId": "1",
            "assetNumber": "1",
            "assetEnName": "",
            "assetName": "name1"
          }
        ]
      }
    ]
  }
}
```

* 失败

```json
{
  "code": 501,
  "status": "failed",
  "msg": "获取企业列表和资产信息失败",
  "data": null
}
```

## 5. 数据结构

### Enterprise

| 参数名称                | 类型                         | 描述   |
|---------------------|----------------------------|------|
| participantId          | string                     | 平台ID |
| entityName      | string                     | 实体名称 |
| enterpriseAssetList | array of [EnterpriseAsset] | 资产列表 |

### EnterpriseAsset

| 参数名称            | 类型     | 描述                 |
|-----------------|--------|--------------------|
| assetId         | string | 主键ID               |
| assetNumber     | string | 资产编号               |
| assetEnName     | string | 资产英文名              |
| assetName       | string | 资产名称               |
| dataProductType | int32  | 数据资产类型：0数据集 |
